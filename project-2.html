Cascaded VGG16 and Xception

A Neural Network model where VGG16 was kept at the input endpoint and its output was fed as input to Xception model. The final output by Xception is then used for prediction. Model was trained on Plant Leaf Dataset.

GitHub Link

Project Overview

Cascaded Machine Learning model (also known as Stacked Models) are used to solve complex problems. The purpose here was to cascade VGG16 and Xception model that are very powerful on their own too. Padding and Pooling Layers were added in between to make sure that ouput of VGG16 can be fed to Xception as input.

Since both models are pretty bulky with lots of training parameter, the layers of VGG16 were prepopulated with imagenet weights and set to be non-trainable except the last layers. This decreases the training time by a large factor. The training was done on Xception instead. To train the model, the plant leaf dataset was split into train and test set with 8:2 ratio.

To further increase the training quality, data augmentation was done on training dataset. Transformation like Shearing, Zooming and Horizontal Flip were performed on the dataset. Final graph were created using Matplotlib.
